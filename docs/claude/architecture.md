# Architecture έΑΦ Coffee Crossword

## Stack decisions

| Layer | Technology | Reason |
|---|---|---|
| Desktop app | Tauri v2 | Native Mac .app, small binary, Rust backend, reuses web UI |
| UI framework | React + TypeScript + Vite | Familiar ecosystem, works in Tauri and browser |
| Search engine | Rust | C++-class performance, compiles to native and WASM |
| Browser target | WASM build of Rust engine | Same code, no JS fallback |
| Mobile (future) | React Native or Flutter | Deferred έΑΦ not in scope yet |
| Styling | TailwindCSS v4 | Utility-first, fast iteration |
| CLI | Rust binary + clap v4 | Same engine, no extra runtime |

### Why not Electron
Electron bundles a full Chromium copy (~200MB app). Tauri uses the OS webview
and a Rust backend, resulting in a ~3έΑΥ10MB app with genuinely native performance.

### Why not Flutter
Flutter ships its own renderer, adding complexity and diverging from web
standards. Tauri lets us ship one web UI everywhere.

## Architecture overview

```
UI Layer (React / App.tsx)
    έΗΧ  Tauri commands (async IPC via invoke()) + Tauri events (listen())
Rust Backend (lib.rs)
    έΑΦ app state (registry, cache handles)
    έΑΦ native menu bar construction
    έΑΦ menu event έΗΤ frontend event bridge
    έΑΦ exposes: search, get_registry, set_active_lists, build_list_cache,
               set_dedup_enabled, rename_list, validate_pattern, describe_pattern
    έΗΧ
Cache Layer (cache.rs)
    έΑΦ builds .tsc from .txt
    έΑΦ memory-maps .tsc for zero-heap-allocation word access
    έΑΦ CacheHandle exposes iter_by_norm_len(), entry_count()
    έΗΧ
Registry (registry.rs)
    έΑΦ scans dictionaries/ folder at startup
    έΑΦ tracks per-list CacheState (Ready/NeedsRebuild/NotBuilt/Building/Error)
    έΑΦ persists active_ids ordering and dedup_enabled via tauri-plugin-store
    έΗΧ
Engine (engine/)
    έΑΦ search_cache(cache, pattern, min, max, normalize) έΗΤ Vec<MatchGroup>
    έΑΦ search_words(words, pattern, min, max, normalize) έΗΤ Vec<MatchGroup>
    έΑΦ validate_pattern, describe_pattern, normalize
    έΗΧ
Dedup (dedup.rs)
    έΑΦ deduplicate(results: &mut Vec<ListSearchResult>)
    έΗΧ
CLI (bin/ccli.rs)
    έΑΦ multi-dict, --build-cache, --no-cache, --no-dedup
```
