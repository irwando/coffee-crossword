Console Application Usage Licensed Version Only

The console version of TEA is supplied as the TEACON application. This is normally run from the MS-DOS Prompt or from within a batch file.

TEACON derives its default settings from those in effect for the Windows application. These can be overridden using command line switches:

Option Example Meaning
/H:hash /H:[^aeiou] The # Macro in patterns is replaced by the specified characters
/A:at /A:[aeiou] The @ Macro in patterns is replaced by the specified characters
/S:shortest /S:3 Words no shorter than the specified value are output
/L:longest /L:15 Words no longer than the specified value are output
/N:annotations /N:0 Annotations are shown if the value is 1 and omitted if the value is 0
/B:balances /B:1 Anagram Balances are shown if the value is 1 and omitted if the value is 0
/X:variables /X:0 Letter Variables are exclusive if the value is 1 and non-exclusive if the value is 0
/U:ignored /U:0 Unused punctuation is ignored in Punctuation Matching if the value is 1 and significant if the value is 0
/D:dictionaries /D:custom.tsd Overrides the list of TEA&Sympathy Dictionary File to be searched; use semi-colons to separate multiple files
/O:output /O:2 This value determines how matched output is presented:
0 = exactly as in the dictionary
1 = as ASCII
2 = as ASCII letters
3 = as ASCII lower case letters
4 = as ASCII upper case letters
/Q /Q Sets quiet mode (patterns and comments aren't echoed)

You can specify the patterns to be matched in two different ways:

  • patterns specified on the command line are tried in sequence; in this mode, the standard input is ignored
  • if no patterns are specified on the command line, each line of standard input is assumed to be a pattern to be matched, except that lines starting with white space are echoed to standard output and otherwise ignored (they can therefore be used for comments)

Examples

TEACON /S:4 /L:8 *J *Q outputs words of between 4 and 8 letters ending J, then those ending Q
TEACON /O:4 < patterns.txt output words matching each pattern specified in patterns.txt in turn, presenting the results in upper case ASCII
